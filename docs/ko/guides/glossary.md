# 용어집

- **주체 (Subject)**: 권한을 부여받을 수 있는 엔티티——사용자, 서비스 계정, 익명 방문자.
- **권한 (Permission)**: 권한 부여의 최소 단위——리소스에 대한 이름 지정 작업.
- **역할 (Role)**: 주체에 할당되는 권한의 이름 지정 컬렉션.
- **RBAC0**: 기본 RBAC 모델: 주체 → 역할 → 권한.
- **RBAC1**: 계층형 RBAC —— 역할이 부모 역할로부터 권한 상속.
- **RBAC2**: 제약 RBAC —— 직무 분리 및 기타 정책 강제.
- **SSD**: 정적 직무 분리 —— 충돌 역할을 동일 사용자에 할당 불가.
- **DSD**: 동적 직무 분리 —— 충돌 역할을 동일 세션에서 활성화 불가.
- **거부 우선 (Deny-override)**: 거부된 권한이 부여된 권한보다 우선.
- **아이덴티티 (Identity)**: 사용자의 인증 신원 유형 (익명, 기본, 임시, 서비스).
- **자격 증명 (Credential)**: 신원 증명——비밀번호, JWT, OAuth 토큰 등.
- **패스포트 (Passport)**: 인증을 위한 챌린지-응답 메커니즘.
- **세션 (Session)**: 주체와 활성 역할이 있는 인증된 컨텍스트.
- **추가 권한 (Extra Permissions)**: 역할 변경 없는 임시 권한 상승.
- **거부 권한 (Denied Permissions)**: 모든 역할 기반 부여를 무시하는 블랙리스트.
- **감사 로거 (Audit Logger)**: 규정 준수를 위해 모든 권한 확인 결과를 기록.
- **권한 레지스트리 (Permission Registry)**: 시스템의 모든 알려진 권한 목록.
- **역할 레지스트리 (Role Registry)**: 모든 정의된 역할과 그 권한 목록.
- **할당 저장소 (Assignment Store)**: 어느 주체가 어떤 역할을 보유하는지 영구 저장.
- **제약 검증기 (Constraint Validator)**: SSD, DSD, 기수, 전제 조건 제약에 대해 역할 할당 검증.
