# 術語表

- **主體 (Subject)**：可被授予權限的實體——使用者、服務帳戶、匿名訪客。
- **權限 (Permission)**：授權的最小單元——針對資源的命名操作。
- **角色 (Role)**：權限的命名集合，指派給主體。
- **RBAC0**：基礎 RBAC 模型：主體 → 角色 → 權限。
- **RBAC1**：層級 RBAC —— 角色從父角色繼承權限。
- **RBAC2**：約束 RBAC —— 強制實施職責分離及其他策略。
- **SSD**：靜態職責分離 —— 衝突角色不能指派給同一使用者。
- **DSD**：動態職責分離 —— 衝突角色不能在同一工作階段中啟用。
- **拒絕優先 (Deny-override)**：被拒絕的權限優先於授予的權限。
- **身份 (Identity)**：使用者的驗證身份類型（匿名、基本、臨時、服務）。
- **憑證 (Credential)**：身份證明——密碼、JWT、OAuth 權杖等。
- **證件 (Passport)**：用於驗證的挑戰-回應機制。
- **工作階段 (Session)**：帶有主體和啟用角色的已驗證上下文。
- **額外權限 (Extra Permissions)**：無需更改角色的臨時權限提升。
- **拒絕權限 (Denied Permissions)**：覆蓋所有基於角色的授權的黑名單。
- **審計日誌 (Audit Logger)**：記錄所有權限檢查結果以供合規使用。
- **權限註冊表 (Permission Registry)**：系統中所有已知權限的權威清單。
- **角色註冊表 (Role Registry)**：所有已定義角色及其權限的權威清單。
- **指派儲存 (Assignment Store)**：持久化哪些主體持有哪些角色。
- **約束驗證器 (Constraint Validator)**：根據 SSD、DSD、基數和前提約束驗證角色指派。
