# 用語集

- **サブジェクト (Subject)**：パーミッションを付与できるエンティティ——ユーザー、サービスアカウント、匿名訪問者。
- **パーミッション (Permission)**：認可の最小単位——リソースに対する名前付き操作。
- **ロール (Role)**：サブジェクトに割り当てられるパーミッションの名前付きコレクション。
- **RBAC0**：基本 RBAC モデル：サブジェクト → ロール → パーミッション。
- **RBAC1**：階層型 RBAC —— ロールが親ロールからパーミッションを継承。
- **RBAC2**：制約付き RBAC —— 職務分離やその他のポリシーを強制。
- **SSD**：静的職務分離 —— 競合するロールを同じユーザーに割り当て不可。
- **DSD**：動的職務分離 —— 競合するロールを同じセッションでアクティブに不可。
- **拒否優先 (Deny-override)**：拒否されたパーミッションは付与されたパーミッションより優先。
- **アイデンティティ (Identity)**：ユーザーの認証 ID タイプ（匿名、基本、一時、サービス）。
- **クレデンシャル (Credential)**：身分証明——パスワード、JWT、OAuth トークンなど。
- **パスポート (Passport)**：認証のためのチャレンジ・レスポンスメカニズム。
- **セッション (Session)**：サブジェクトとアクティブロールを持つ認証済みコンテキスト。
- **追加パーミッション (Extra Permissions)**：ロール変更なしの一時的な権限昇格。
- **拒否パーミッション (Denied Permissions)**：すべてのロールベースの付与を上書きするブラックリスト。
- **監査ロガー (Audit Logger)**：コンプライアンスのためにすべてのパーミッションチェック結果を記録。
- **パーミッションレジストリ (Permission Registry)**：システム内のすべての既知パーミッションの正規リスト。
- **ロールレジストリ (Role Registry)**：すべての定義済みロールとそのパーミッションの正規リスト。
- **割り当てストア (Assignment Store)**：どのサブジェクトがどのロールを保持するかを永続化。
- **制約バリデータ (Constraint Validator)**：SSD、DSD、基数、前提制約に対してロール割り当てを検証。
