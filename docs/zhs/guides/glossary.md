# 术语表

- **主体 (Subject)**：可被授予权限的实体——用户、服务账户、匿名访客。
- **权限 (Permission)**：授权的最小单元——针对资源的命名操作。
- **角色 (Role)**：权限的命名集合，分配给主体。
- **RBAC0**：基础 RBAC 模型：主体 → 角色 → 权限。
- **RBAC1**：层级 RBAC —— 角色从父角色继承权限。
- **RBAC2**：约束 RBAC —— 强制实施职责分离及其他策略。
- **SSD**：静态职责分离 —— 冲突角色不能分配给同一用户。
- **DSD**：动态职责分离 —— 冲突角色不能在同一会话中激活。
- **拒绝优先 (Deny-override)**：被拒绝的权限优先于授予的权限。
- **身份 (Identity)**：用户的认证身份类型（匿名、基本、临时、服务）。
- **凭证 (Credential)**：身份证明——密码、JWT、OAuth 令牌等。
- **凭据 (Passport)**：用于认证的挑战-响应机制。
- **会话 (Session)**：带有主体和活动角色的已认证上下文。
- **额外权限 (Extra Permissions)**：无需更改角色的临时权限提升。
- **拒绝权限 (Denied Permissions)**：覆盖所有基于角色的授权的黑名单。
- **审计日志 (Audit Logger)**：记录所有权限检查结果以供合规使用。
- **权限注册表 (Permission Registry)**：系统中所有已知权限的权威列表。
- **角色注册表 (Role Registry)**：所有已定义角色及其权限的权威列表。
- **分配存储 (Assignment Store)**：持久化哪些主体持有哪些角色。
- **约束验证器 (Constraint Validator)**：根据 SSD、DSD、基数和前提约束验证角色分配。
