# Глоссарий

- **Субъект (Subject)**: Сущность, которой могут быть предоставлены разрешения — пользователь, сервисная учётная запись, анонимный посетитель.
- **Разрешение (Permission)**: Атомарная единица авторизации — именованное действие над ресурсом.
- **Роль (Role)**: Именованный набор разрешений, назначаемый субъектам.
- **RBAC0**: Базовая модель RBAC: Субъект → Роль → Разрешение.
- **RBAC1**: Иерархический RBAC — роли наследуют разрешения от родительских ролей.
- **RBAC2**: Ограниченный RBAC — обеспечивает разделение обязанностей и другие политики.
- **SSD**: Статическое разделение обязанностей — конфликтующие роли не могут быть назначены одному пользователю.
- **DSD**: Динамическое разделение обязанностей — конфликтующие роли не могут быть активны в одной сессии.
- **Приоритет отказа (Deny-override)**: Отказанные разрешения имеют приоритет над предоставленными.
- **Идентичность (Identity)**: Тип аутентификационной идентичности пользователя (Анонимный, Базовый, Временный, Сервисный).
- **Учётные данные (Credential)**: Доказательство личности — пароль, JWT, токен OAuth и т.д.
- **Паспорт (Passport)**: Механизм запрос-ответ для аутентификации.
- **Сессия (Session)**: Активный аутентифицированный контекст с субъектом и активными ролями.
- **Дополнительные разрешения (Extra Permissions)**: Временное повышение разрешений без изменения роли.
- **Отказанные разрешения (Denied Permissions)**: Чёрный список, переопределяющий все ролевые предоставления.
- **Аудиторский журнал (Audit Logger)**: Записывает все результаты проверок разрешений для соответствия требованиям.
- **Реестр разрешений (Permission Registry)**: Авторитетный список всех известных разрешений в системе.
- **Реестр ролей (Role Registry)**: Авторитетный список всех определённых ролей и их разрешений.
- **Хранилище назначений (Assignment Store)**: Сохраняет, какие субъекты имеют какие роли.
- **Валидатор ограничений (Constraint Validator)**: Проверяет назначения ролей на соответствие ограничениям SSD, DSD, кардинальности и предварительным условиям.
